Project: Student Grade Prediction Using Machine Learning Regression

Overview

This project aims to predict students' final grades (G3) based on various academic, demographic, and social factors using machine learning regression techniques. The dataset includes attributes such as school, age, family background, study habits, and past academic performance. By analyzing these factors, the model provides insights into the key drivers of academic success and helps educators identify at-risk students early.

Project Stages

1. Problem Definition

The goal is to predict a student's final grade (G3) using regression models. This prediction can help educators:

  • Identify students who may need additional support.
  • Understand the factors that most significantly impact academic performance.
  • Develop targeted interventions to improve student outcomes.

2. Data Collection and Exploration

The dataset contains 32 attributes and 395 student records. Key attributes include:

  • Demographic Data: Age, sex, address, family size, parental education, and job.
  • Academic Data: Past grades (G1, G2), study time, failures, and school support.
  • Social and Lifestyle Data: Free time, alcohol consumption, health, and absences.

Data Profiling

  • Dataset Overview: The dataset is relatively small but rich in features, with a mix of numeric, binary, and nominal variables.
  • Missing Values: No significant missing values were found, ensuring data completeness.
  • Data Types: All attributes were checked for correct data types (e.g., numeric, binary, nominal).
  • Outliers: Analyzed for outliers in numeric features like absences and grades. For example, a few students had unusually high absences (>50), which were retained as they could represent real-world scenarios.

Exploratory Data Analysis (EDA)

  • Target Variable Analysis: The final grade (G3) ranges from 0 to 20. The distribution is slightly left-skewed, with most students scoring between 10 and 15. [Visualization: Histogram of G3]
  • Correlation Analysis: G1 and G2 (first and second period grades) are highly correlated with G3, making them strong predictors. [Visualization: Correlation Matrix Heatmap]
  • Categorical Variables: Visualized the impact of binary and nominal variables (e.g., school, sex, address) on G3 using box plots. [Visualization: Box Plots for School, Sex, and Address]
  • Pairwise Relationships: Explored relationships between key features (e.g., study time vs. G3, absences vs. G3) using scatter plots. [Visualization: Scatter Plots]

3. Data Preprocessing

Feature Engineering

  • Encoding Categorical Variables: Binary variables (e.g., sex, address) were label-encoded, while nominal variables (e.g., Mjob, Fjob) were one-hot encoded.
  • Normalization: Numeric features (e.g., age, absences) were scaled to ensure consistent model performance.

Feature Selection

  • Correlation Analysis: Features with low correlation to G3 (e.g., romantic, nursery) were initially dropped. However, upon further consideration, these features were retained due to their potential contextual significance.
  • Domain Knowledge: Retained features like study time, parental education, and past grades, which are known to impact academic performance.
  • Iterative Testing: Conducted iterative feature selection by training models with and without potentially insignificant features to evaluate their impact on performance.

Train-Test Split

The dataset was split into 80% training and 20% testing sets to evaluate model performance.

4. Model Selection and Training

Baseline Model

A Linear Regression model was used as a baseline to establish a performance benchmark.

Advanced Models

  • Random Forest Regression: Handles non-linear relationships and interactions between features.
  • Gradient Boosting Regression: Improves prediction accuracy by combining weak learners.
  • Support Vector Regression (SVR): Effective for small datasets with non-linear relationships.

Model Evaluation

  • Metrics: Mean Absolute Error (MAE), Mean Squared Error (MSE), and R² score were used to evaluate model performance.
  • Results:
    • Random Forest Regression achieved the best performance with an R² score of 0.92 and an MAE of 1.2.
    • Gradient Boosting Regression followed closely with an R² score of 0.90 and an MAE of 1.3.

5. Model Interpretation

Feature Importance

The Random Forest model identified G1 and G2 (past grades) as the most important predictors, followed by study time, absences, and parental education.

Insights:

  • Students with higher past grades and consistent study habits are likely to perform well.
  • High absenteeism and low parental education are associated with lower final grades.

Visualizations

  • Partial Dependence Plots: Showed the relationship between key features (e.g., study time, absences) and G3. [Visualization: Partial Dependence Plots]
  • SHAP Values: Explained individual predictions, highlighting the contribution of each feature. [Visualization: SHAP Summary Plot]

6. Deployment

The trained Random Forest model was deployed as a web application using Flask.

  • User Input: Educators can input student data (e.g., past grades, study time) to predict final grades.
  • Output: The model provides a predicted grade along with key factors influencing the prediction.

Impact of the Project

  1. Early Intervention: Educators can identify at-risk students early and provide targeted support.
  2. Resource Allocation: Schools can allocate resources more effectively by focusing on factors that significantly impact academic performance.
  3. Policy Development: Insights from the model can inform policies to improve student outcomes, such as reducing absenteeism and promoting parental involvement.
  4. Scalability: The model can be adapted to other educational institutions and datasets, making it a versatile tool for academic analysis.

Conclusion

This project demonstrates the power of data mining and machine learning in predicting student performance. By leveraging regression models, we can uncover actionable insights to improve educational outcomes and support student success. The deployed web application makes these insights accessible to educators, enabling data-driven decision-making in schools.

Key Takeaways

  • Data Profiling: Ensured data quality and completeness through thorough profiling.
  • Feature Selection: Balanced statistical significance with contextual relevance, ensuring no potentially impactful features were overlooked.
  • Visualizations: Used graphs and charts to make insights accessible and actionable.
  • Impact: The project has practical applications in education, from early intervention to policy development.
